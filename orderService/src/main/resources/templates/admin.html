🧪 테스트 시나리오
아래 시나리오를 순서대로 따라하면 핵심 기능을 한 번에 검증할 수 있습니다.
SCENARIO 1 동시 주문 · 보상 트랜잭션 핵심
▸ 목적
Redis 원자적 재고 선차감 + 결제 실패 시 보상 트랜잭션 확인
▸ 준비
탭 A · 탭 B : /orderView  (탭 B는 시크릿 탭 — 별도 uuid)
탭 C : /admin
▸ 실행 순서
1. 탭 C → 재고 "한정판 세트" +2 조정 (기본 1 → 총 3개)
2. 탭 A: 한정판 세트 2개 담기 → 결제 진행
3. 탭 B: 동시에 한정판 세트 2개 담기 → 결제 진행
▸ 기대 결과
✅ 서버 수신 순서 기준 먼저 온 쪽만 재고 점유
✅ 늦은 쪽: releaseStock() → Redis 재고 복구
✅ 늦은 쪽: DB CANCELLED · 고객 화면 실패 배너
✅ 탭 C 재고 수치 즉시 차감 반영
SCENARIO 2 admin 브로드캐스트 · 수평확장
▸ 목적
Fanout Exchange → 다수 admin 화면 동기화 + 수평확장 구조 확인
▸ 준비 및 실행
탭 A · 탭 B : /orderView
탭 C · 탭 D : /admin (2개 동시 열기)
탭 A · B 순서대로 주문 진행
▸ 기대 결과
✅ 탭 C · D 양쪽에 세션카드 동시 추가됨
✅ admin.notify.exchange Fanout 정상 동작 확인
✅ ECS Task 수평확장 시에도 동일 구조 유지
SCENARIO 3 세션 생명주기 · 추가 주문
▸ 목적
조리 대기 중 추가 주문 시 동일 SSE 연결 유지 확인
▸ 실행 순서
1. 짜장면 1개 주문 → COOKING_START 수신 확인
2. 조리 완료 전(7초 이내) 짬뽕 추가 주문
3. 두 번의 COOKING_DONE 수신 확인
▸ 기대 결과
✅ 추가 주문 시 SSE 재연결 없음 (동일 연결 유지)
✅ Heartbeat 30초마다 Redis TTL 갱신
✅ 모든 주문 수령 후 SSE 연결 종료
SCENARIO 4 수령 완료 · 세션 소멸
▸ 목적
수령 완료 이벤트 전파 및 양쪽 화면 세션 정리 확인
▸ 실행 순서
1. 고객: 주문 완료 → 조리완료(COOKING_DONE) 대기
2. 관리자: 세션카드의 '수령' 버튼 클릭
▸ 기대 결과
✅ 고객 화면: 수령 완료 표시 → 10초 후 자동 초기화
✅ admin: 세션카드 페이드아웃 후 제거
✅ Redis 세션 TTL 만료 정리
이 프로젝트에 대해
🎯 만든 이유
팝업 푸드트럭 환경에서 회원 가입 없이 QR 하나로 주문부터 수령까지 끊김없이 처리하는 시스템이 필요했습니다. 동시에 Redis 재고 관리, RabbitMQ 비동기 통신, SSE 실시간 알림, MSA 구조 등 실제 트래픽이 발생하는 환경에서 각 기술이 어떻게 맞물려 동작하는지를 직접 구현하고 검증하기 위한 토이 프로젝트입니다.
🔑 핵심 기술 검증 포인트
Redis DECR 원자 연산 — 동시 주문에서 재고가 음수로 떨어지지 않는가
보상 트랜잭션 — 결제 실패 시 선차감된 재고가 정확히 복구되는가
SSE + Heartbeat — 조리 완료 알림이 연결 끊김 없이 고객에게 전달되는가
Fanout 브로드캐스트 — ECS Task가 늘어도 모든 관리자 화면이 동기화되는가
📌 의도적으로 단순화한 부분
회원/인증 없음 (UUID 익명 세션) · 조리 시간 7초 자동 완성 · 기본 현금 고객 2명 상시 존재 · RDS/ElastiCache 대신 EC2 컨테이너 운영 (비용 절감)

트러블슈팅 기록
개발 · 배포 과정에서 마주친 문제와 해결 과정을 기록합니다.
✗ SSE 재연결 시 신규 emitter가 즉시 cleanup 됨
원인  old.complete() 비동기 콜백이 새 emitter를 map에서 제거
해결  complete() 제거 + 콜백 등록 시 동일성(key) 체크 추가
✗ 실시간 재고 갱신 안 됨
원인  AdminNotifyConsumer payload를 Map<String,String>으로 캐스팅
해결  Map<String,Object>로 변경
✗ 만료된 uuid SSE 재연결 차단
원인  exists() false → 연결 거부
해결  만료 uuid 재등록 후 허용 (Heartbeat로 TTL 지속 갱신)
✗ ECS → EC2 DB 연결 실패
원인  application-prod.yml에 퍼블릭 IP 하드코딩
해결  CloudFormation Export 값(프라이빗 IP)을 CI/CD에서 동적으로 주입
✗ MySQL 한글 데이터 깨짐 (latin1)
원인  Docker MySQL 컨테이너 기본 charset이 latin1
해결  my.cnf 볼륨 마운트로 utf8mb4 강제 적용 + 컨테이너 재생성
✗ admin SSE 연결 요청 자체가 안 보내짐
원인  crypto.randomUUID()는 HTTPS/localhost에서만 동작 — ALB HTTP 환경에서 에러
해결  HTTP 환경 폴백 UUID 생성 로직 추가
✗ 로컬 H2 JPA 초기화 순서 오류
원인  defer-datasource-initialization: true 누락으로 schema.sql이 JPA보다 먼저 실행됨
해결  application-local.yml에 해당 설정 추가
